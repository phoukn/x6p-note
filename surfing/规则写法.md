| 类型                  | 写法示例                                      | 说明                                                                 | 常用目标 / 平台          |
|-----------------------|-----------------------------------------------|----------------------------------------------------------------------|--------------------------|
| 完整域名              | `DOMAIN,www.example.com,Proxy`                | 完全匹配域名（不含协议）                                             | —                        |
| 域名后缀              | `DOMAIN-SUFFIX,google.com,Proxy`              | **最常用**，匹配域名后缀（不需加点）                                 | Proxy / DIRECT           |
| 域名关键字            | `DOMAIN-KEYWORD,google,Proxy`                 | 域名任意位置包含关键词                                               | —                        |
| 域名通配符            | `DOMAIN-WILDCARD,*.google.com,Proxy`          | 支持 * 通配符（较少用）                                              | —                        |
| 域名正则              | `DOMAIN-REGEX,^www\\..+\\.google\\.com$,Proxy` | 正则匹配（性能稍差，慎用）                                           | 高级场景                 |
| IP-CIDR (IPv4/IPv6)   | `IP-CIDR,192.168.0.0/16,DIRECT,no-resolve`    | CIDR 网段匹配，可加 `no-resolve`（推荐局域网用）                     | DIRECT / REJECT          |
| IP-CIDR6              | `IP-CIDR6,fc00::/7,DIRECT`                    | 专门 IPv6 CIDR                                                       | DIRECT                   |
| GEOIP                 | `GEOIP,CN,DIRECT`                             | 国家/地区代码（CN 最常用）                                           | DIRECT                   |
| GEOSITE               | `GEOSITE,cn,DIRECT`                           | 域名分类集（cn、gfw、google、netflix 等）                            | DIRECT / Proxy           |
| RULE-SET              | `RULE-SET,geosite-google,Proxy`               | 引用外部规则集（.yaml / .text，最现代主流）                          | **现代主流**             |
| **进程名匹配** (PROCESS-NAME) | `PROCESS-NAME,chrome.exe,Proxy`               | 匹配进程名；**Android 上可直接用包名** 如 `com.twitter.android`      | Android/Windows/Mac 专用 |
| **进程名通配**        | `PROCESS-NAME-WILDCARD,*telegram*,Proxy`      | 支持 * ? 通配符；Android 也支持包名通配                              | Android 常用             |
| **进程名正则**        | `PROCESS-NAME-REGEX,(?i)Telegram.*,Proxy`     | 正则匹配进程名/包名（不区分大小写用 (?i)）                           | 高级分流                 |
| **进程路径匹配**      | `PROCESS-PATH,/usr/bin/wget,Proxy`            | 完整进程路径匹配（桌面系统常用）                                     | Windows/Mac/Linux 专用   |
| **进程路径通配**      | `PROCESS-PATH-WILDCARD,C:\*\chrome.exe,Proxy` | 通配进程路径                                                         | —                        |
| **进程路径正则**      | `PROCESS-PATH-REGEX,.*\\Chrome\\Application\\chrome.exe,Proxy` | 正则进程路径                                                         | 高级                     |
| 逻辑组合              | `AND,((GEOIP,CN),(NETWORK,UDP)),REJECT`       | 与/或/非 组合（AND/OR/NOT）                                          | 特殊场景（如 UDP 直连）  |
| MATCH（兜底）         | `MATCH,Proxy`                                 | 所有未匹配规则的默认出口                                             | 必备                     |
